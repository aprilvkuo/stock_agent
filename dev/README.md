# 🚧 草稿区 (Development Area)

**职责**: 开发中、实验性、未完成的代码

---

## 📁 目录结构

```
dev/
├── experiments/    # 实验性代码
├── drafts/         # 草稿文件
└── testing/        # 测试代码
```

---

## 📋 使用规范

### ✅ 允许

- 实验性代码
- 未完成的草稿
- 测试代码
- 原型验证

### ❌ 禁止

- 生产代码
- 已验证的功能（应迁移到正式目录）
- 大型数据文件

---

## 🔄 迁移流程

1. 在 dev/ 中开发和测试
2. 验证通过后
3. 迁移到正式目录（memory/, skills/, agents/）
4. Git 提交
5. 清理 dev/ 中的旧文件
