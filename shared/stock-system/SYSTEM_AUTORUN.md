## 🔄 自动化进化流程

### 日常循环（每日自动）

```
┌─────────────────────────────────────────────────────────┐
│                    每日自动化循环                        │
└─────────────────────────────────────────────────────────┘

00:00 ─┬─ 守护进程持续运行（检查请求）
       │
09:00 ─┼─ 预测验证（auto_agent.py daily）
       │   └─ 验证到期预测 → 更新准确率 → 调整权重
       │
15:00 ─┼─ 质检 Agent 深度检查
       │   └─ 分析错误趋势 → 创建工单（如有问题）
       │
20:00 ─┼─ 程序员 Agent 代码优化
       │   └─ 检查工单 → 修复 Bug → 优化性能
       │
24:00 ─┴─ 日志归档
           └─ 备份今日日志 → 清理旧日志
```

### 周度循环（每周五自动）

```
┌─────────────────────────────────────────────────────────┐
│                    每周自动化循环                        │
└─────────────────────────────────────────────────────────┘

周五 20:00 ─┬─ 周度复盘（auto_agent.py weekly）
            │
            ├─ 1. 统计本周分析记录
            │   └─ 各 Agent 分析次数、准确率
            │
            ├─ 2. 计算 Agent 表现
            │   └─ 准确率排名、权重调整建议
            │
            ├─ 3. 生成周度报告
            │   └─ reports/weekly-YYYY-MM-DD.md
            │
            ├─ 4. 提炼有效规则
            │   └─ 从验证数据中提取成功模式
            │
            └─ 5. 更新知识库
                └─ 写入 knowledge/ 目录
```

### 月度循环（每月 1 号自动）

```
┌─────────────────────────────────────────────────────────┐
│                    每月自动化循环                        │
└─────────────────────────────────────────────────────────┘

每月 1 号 ─┬─ 月度考核
           │
           ├─ 1. 统计各 Agent 月度表现
           │   └─ 准确率、错误率、奖惩记录
           │
           ├─ 2. 执行奖惩
           │   └─ 月度最佳 → 奖励
           │   └─ 月度末位 → 惩罚
           │
           ├─ 3. 权重调整
           │   └─ 根据准确率动态调整
           │
           ├─ 4. 生成月度报告
           │   └─ reports/monthly-YYYY-MM.md
           │
           └─ 5. 系统优化建议
               └─ 识别瓶颈 → 创建优化工单
```

## 📈 进化机制

### 1. 数据驱动进化

```
分析 → 预测 → 验证 → 反馈 → 优化
  ↓      ↓       ↓       ↓       ↓
Agent  写入队列  到期检查  准确率  调整权重
```

**关键文件**:
- `validation-queue.md` - 待验证预测
- `analysis-log/` - 分析历史记录
- `reports/` - 验证报告

**进化周期**: 30 天（首批验证日：2026-04-07）

---

### 2. 工单驱动修复

```
质检发现 → 创建工单 → 程序员修复 → 验证效果
   ↓          ↓           ↓           ↓
错误检测   标准化格式   自动修复    下次质检验证
```

**关键文件**:
- `tickets/` - 工单目录
- `agent-programmer.py` - 程序员 Agent 脚本

**修复周期**: 10 分钟（工单检查频率）

---

### 3. 知识库积累

```
成功经验 → 提炼规则 → 写入知识库 → 应用规则
   ↓           ↓           ↓           ↓
验证正确   模式识别   knowledge/   下次分析
```

**关键文件**:
- `knowledge/error-patterns.md` - 错误模式
- `knowledge/success-patterns.md` - 成功模式（待创建）
- `knowledge/trading-rules.md` - 交易规则（待创建）

**更新周期**: 每周（周度复盘时）

---

### 4. 动态权重调整

```
准确率统计 → 权重计算 → 应用新权重 → 验证效果
    ↓            ↓           ↓           ↓
每周统计    表现好 +0.05  主 Agent 使用  下周验证
```

**权重范围**:
| Agent | 初始 | 最低 | 最高 |
|-------|------|------|------|
| 基本面 | 0.35 | 0.20 | 0.45 |
| 技术面 | 0.30 | 0.20 | 0.45 |
| 情绪面 | 0.35 | 0.20 | 0.45 |

**调整周期**: 每周（根据周度复盘）
