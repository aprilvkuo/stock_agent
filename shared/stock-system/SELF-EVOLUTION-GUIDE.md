## ⚙️ 工作原理

### 触发条件

系统会在以下情况自动执行自我进化：

1. **空闲触发** (主要方式)
   - 连续 3 次检查没有待处理请求 (约 3 分钟)
   - 自动启动自我进化流程

2. **定时触发** (备份方式)
   - 每 60 分钟自动执行一次
   - 确保即使有任务也会定期进化

### 执行流程

```
┌─────────────────────────────────────┐
│   守护进程检查请求队列              │
└────────────┬────────────────────────┘
             │
    ┌────────┴────────┐
    │  有待处理请求？ │
    └────┬──────┬─────┘
         │      │
        是      否
         │      │
         │      ▼
         │  ┌────────────────────┐
         │  │ 空闲计数器 +1      │
         │  └────────┬───────────┘
         │           │
         │           ▼
         │  ┌────────────────────┐
         │  │ 连续 3 次空闲？      │
         │  └────┬──────┬────────┘
         │       │      │
         │      否      是
         │       │      │
         │       │      ▼
         │       │  ┌──────────────┐

         │       │
         ▼       ▼

    │  继续监控循环          │
    └────────────────────────┘
```

## 📊 自我进化任务详情

### 任务 1: 复盘历史分析结果

**扫描目录**: `/queue/results/`

**统计内容**:
- 总报告数量
- 各 Agent 报告分布 (基本面/技术面/情绪面)
- 覆盖的股票数量

**价值**: 了解系统的工作量和覆盖范围

---

### 任务 2: 分析 Agent 性能

**分析日志**: `/logs/daemon-YYYY-MM-DD.log`

**统计指标**:
- 成功执行次数
- 失败执行次数
- 成功率百分比
- 常见错误类型

**价值**: 识别系统瓶颈和改进点

---

### 任务 3: 更新知识库

**生成文件**: `/knowledge/stock-wisdom.md`

**包含内容**:
- 数据源优先级
- 重点股票池
- 分析参数优化建议
- 常见错误处理方案
- 改进建议

**价值**: 积累系统知识，避免重复错误

---

### 任务 4: 生成改进计划

**计划分类**:
1. **立即执行** - 已完成或正在进行的改进
2. **短期计划** - 接下来要实现的功能
3. **长期计划** - 未来发展方向

**价值**: 明确系统演进方向

## ❓ 常见问题

### Q: 自我进化会影响正常分析任务吗？
**A**: 不会！自我进化只在空闲时执行，有任务时会优先处理任务。

### Q: 进化过程需要多长时间？
**A**: 通常 1-5 秒，取决于历史数据量。

### Q: 可以手动触发进化吗？
**A**: 可以！运行 `python3 self-evolution.py` 即可。

### Q: 进化报告会占用很多空间吗？
**A**: 不会，每份报告约 1-5KB，建议定期归档。

### Q: 如何知道系统是否在进化？
**A**: 查看日志中的 "🧬 开始自我进化" 和 "✅ 自我进化完成" 记录。
