- [ ] 问题定位 - 分析具体原因，找出 root cause

## 处理流程

1. 接收任务 - @programmer-agent 确认接单

3. 解决问题 - 快速迭代，及时同步进度
4. 创建 PR - 关联此 Issue (Closes #)
5. Code Review - 等待 @qa-agent 审核
6. Merge 关闭 - 合并后自动关闭

## 特点总结

### 优势
- 纯文字，无 emoji
- 互联网风格用词（接单、迭代、闭环、root cause）
- 表格化信息展示
- 清晰的 checklist
- 明确的流程说明
- 预留沟通区域

### 语气
- 专业但不生硬
- 现代化技术团队风格
- 有温度但不随意
