## 📋 核心流程

```
┌─────────────────────────────────────────────────────────────┐
│                    五步工作法                                │
│                                                             │
│  1️⃣ UPDATE → 2️⃣ READ → 3️⃣ DO → 4️⃣ CHECK → 5️⃣ REVIEW      │
│     ↓          ↓         ↓        ↓          ↓              │
│   更新       读取      执行     检查       复盘             │
│   信息       清单      任务     结果       优化             │
└─────────────────────────────────────────────────────────────┘
```

---

## 1️⃣ UPDATE - 更新当前状态

**目的**: 确保信息同步，避免基于过期信息做决策

**执行内容**:
```markdown
- [ ] 更新系统版本信息
- [ ] 更新当前能力清单
- [ ] 更新任务解决进度
- [ ] 更新待办事项状态
- [ ] 记录当前时间戳
```

**输出示例**:
```markdown
## 当前状态 (2026-03-08 14:32)

**系统版本**: v2.2
**当前能力**:
  ✅ 多 Agent 分析
  ✅ Agent 协作
  ✅ 数据缓存
  ✅ 监控网站
  ⏳ 程序员 Agent (规划中)

**进行中任务**:
  - v2.3 开发 (进度：0%)

**待办任务**: 3 项（见 TODO.md）
```

**位置**: `/Users/egg/.openclaw/workspace/STATUS.md`

---

## 2️⃣ READ - 读取 TODO 清单

**目的**: 明确任务优先级，避免遗漏和重复工作

**执行内容**:
```markdown
- [ ] 读取 TODO.md
- [ ] 识别高优先级任务
- [ ] 确认任务依赖关系
- [ ] 估算工作量
- [ ] 选择本次执行任务
```

**输出示例**:
```markdown
## 本次任务选择

**高优先级任务**:
1. 🔴 复盘 Agent 增强（30 分钟）
2. 🔴 创建程序员 Agent（1 小时）
3. 🔴 工单系统实现（30 分钟）

**本次选择**: 任务 1 - 复盘 Agent 增强
**理由**: 
  - 优先级最高
  - 工作量适中
  - 是其他任务的基础

**预计完成时间**: 14:32 - 15:02
```

**位置**: TODO.md

---

## 3️⃣ DO - 执行任务

**目的**: 高质量完成任务，同时记录过程

**执行内容**:
```markdown
- [ ] 制定实施计划
- [ ] 分步骤执行
- [ ] 记录关键决策
- [ ] 保存中间结果
- [ ] 遇到问题及时记录
```

**执行模板**:
```markdown
## 任务执行记录

**任务**: 复盘 Agent 增强
**开始时间**: 14:32
**结束时间**: 15:02

### 步骤 1: 需求分析
- 分析内容：...
- 决策：...

### 步骤 2: 实施
- 修改文件：...
- 新增代码：...

### 步骤 3: 测试
- 测试用例：...
- 测试结果：...

### 遇到的问题
1. 问题描述 → 解决方案
2. 问题描述 → 解决方案
```

**位置**: 任务执行日志（按日期保存）

---

## 4️⃣ CHECK - 自助检查

**目的**: 确保任务完成质量，发现潜在问题

**执行内容**:
```markdown
- [ ] 验证任务是否完成
- [ ] 测试功能是否正常
- [ ] 检查是否引入新问题
- [ ] 验证性能是否下降
- [ ] 确认文档是否更新
```

**检查清单**:
```markdown
## 任务完成检查

### 功能检查
- [ ] 核心功能是否正常
- [ ] 边界情况是否处理
- [ ] 错误处理是否完善

### 回归检查
- [ ] 原有功能是否受影响
- [ ] 性能是否下降
- [ ] 是否有新 Bug

### 文档检查
- [ ] 代码注释是否完整
- [ ] 文档是否更新
- [ ] TODO.md 是否标记完成

### 测试结果
- 测试用例数：X
- 通过数：Y
- 失败数：Z
- 通过率：Y/X%
```

**输出示例**:
```markdown
## 检查结果 ✅

**任务完成度**: 100%
**测试结果**: 5/5 通过
**回归测试**: 无问题
**文档更新**: ✅ 已完成

**新问题**: 无
**性能影响**: 无显著影响
```

## 5️⃣ REVIEW - 复盘优化

**目的**: 持续改进，积累经验，规划下一步

**执行内容**:
```markdown
- [ ] 总结本次任务得失
- [ ] 记录经验教训
- [ ] 识别改进点
- [ ] 提出新 TODO
- [ ] 更新 TODO.md
```

**复盘模板**:
```markdown

### 做得好的
1. ...
2. ...

### 需要改进的
1. ...
2. ...

### 经验教训
- 经验 1: ...
- 经验 2: ...

### 新发现的 TODO
1. [ ] 新任务 1（优先级：高/中/低）
2. [ ] 新任务 2（优先级：高/中/低）

### 下一步计划
- 短期（本周）: ...
- 中期（下周）: ...
- 长期（本月）: ...
```

**位置**: 复盘报告 + TODO.md

## 🎯 违规处理

**如果未遵循五步工作法**:

1. **第一次**: 提醒
2. **第二次**: 警告
3. **第三次**: 暂停任务，重新培训

**例外情况**:
- 🔴 紧急 Bug 修复（可先 DO 后补流程）
- 🔴 系统故障（可跳过 UPDATE 直接 DO）
